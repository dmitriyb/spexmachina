## Execution Order

Checkers run in sequence, not in parallel. Schema validation must pass before structural checks can run (you can't check DAGs if the JSON doesn't parse). However, all checkers run even if earlier ones find errors вҖ” the full error report is always produced.

## Error Accumulation

Each checker appends to a shared `[]ValidationError` slice. No checker short-circuits on errors from previous checkers. The final report contains all violations found across all checks.
